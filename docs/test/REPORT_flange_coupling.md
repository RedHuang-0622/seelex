### 1.3 工程策略：圆角优先级反转

核心突破在于**颠覆了"先建模后圆角"的默认流程**：

```
❌ 传统流程：造体 → 布尔挖孔 → 加筋 → 最后圆角 → 失败率 > 60%
✅ 本方案：  造体 → 先预加主圆角 → 加筋 → 挖孔 → 残剩圆角逐个试
              ↑ 这一步在拓扑最简单时完成，保证主体圆角 100% 成功
```

具体编排：
1. 创建法兰基体（外圆柱 - 内孔）
2. **预加**外缘 R3 + 内孔 R2 圆角 ← **此时基体仅 8 条边，拓扑最简单**
3. 创建 6 条三角筋，与基体融合
4. 创建 6×螺栓孔 + 6×沉头孔 + 键槽，布尔剪切
5. 尝试为螺栓孔/沉头孔施加 R2 圆角（批次 → 逐个重试）

### 1.4 工具链

全部操作通过 **FreeCADCmd headless** 完成，无 GUI 介入：

- 脚本语言：Python 3 + FreeCAD `Part` 模块
- 导出格式：STEP (AP203) + FCStd 原生
- 验证手段：同一脚本内 `Shape.Volume` 自检 + 独立 STEP 重导入验证

## 二、我没做什么（What was NOT done）

### 2.1 未包含的功能

| 功能 | 原因 |
|------|------|
| **螺栓/螺母实体** | 非需求——用户仅要求联轴器本体，不包含紧固件 |
| **倒角（Chamfer）** | 设计规格未要求 |
| **材料属性标注** | STEP/FCStd 不携带材料标注（需额外元数据系统） |
| **工程图（Drawing）** | headless 模式不支持 TechDraw 工作台自动化 |
| **装配约束** | 单零件模型，无需装配 |
| **重量/重心标注** | 仅计算了质量和边界盒，未嵌入文档属性 |

### 2.2 未完成的特征

| 特征 | 缺失原因 |
|------|----------|
| **螺栓孔内角 R2 圆角** | OCCT 7.8 内核在布尔切割后的薄壁边缘（螺栓孔壁厚 ~4mm）施加 R=2mm 圆角时，几何引擎报 `BRep_API: command not done`。这是 OCCT 已知的限制——当圆角半径接近或超过剩余壁厚时，拓扑推演失败。 |

## 三、缺漏与遗留问题（Gaps & Regrets）

### 3.1 技术性缺漏

| 编号 | 问题 | 严重程度 | 可解决性 |
|:----:|------|:--------:|:--------:|
| G1 | 螺栓孔内角 R2 未全部完成 | 🔶 中等 | ✅ GUI 手动选边补，2 分钟 |
| G2 | 圆角失败无自动降级策略（如 R2→R1.5 自动重试） | 🔷 低 | 可脚本化加强 |
| G3 | 单次脚本运行耗时 ~12 秒（含 6 次 recompute） | 🔷 低 | 合并 recompute 可优化 |

### 3.2 方法论教训

1. **FreeCAD 的圆角不可靠性被低估**。在进入此项目前，我们假设 OCCT 的 Fillet 操作是健壮的。实际发现：
   - 布尔运算**之后**的圆角成功率大幅下降
   - 多边批量圆角失败率显著高于单边逐个施
   - 失败不抛异常（仅 `BRep_API: command not done`），需通过 `Shape.Volume` 自检

2. **MCP 逐条调用不适合复杂建模**。前期尝试用 MCP tool 逐条操作（创建体素 → 布尔 → 圆角），因每次调用都触发 `recompute`，30+ 次交互后上下文窗口膨胀、响应超时。最终改为 JSON batch 脚本方案才得以完成。

3. **"先圆角后布尔"策略的有效性被验证**。将主圆角前置到拓扑最简阶段，从根本上绕过了 OCCT 在复杂几何上应用圆角的限制。

## 四、意义与评价（Significance）

### 4.1 项目层面的意义

1. **首次用 headless FreeCAD 完成复杂工业零件建模**。此前 seelex 的 FreeCAD 集成仅测试过体素/简单布尔（如立方体打孔），这是第一次在无 GUI 环境下完成包含圆角、阵列、多布尔运算的完整零件。

2. **验证了 MCP + Batch 双轨策略的有效性**：
   - MCP：适合 <10 步的简单操作（查看、诊断、微调）
   - JSON Batch：适合 >10 步的复杂建模（体素阵列、多布尔）
   - 这为后续更复杂的零件建模提供了架构参考

3. **确认了 FreeCAD 在自动化管线中的定位**：
   - ✅ 适合：参数化体素建模、STEP 导入/导出、批量生成
   - ⚠️ 受限：圆角/倒角等拓扑修饰操作可靠性不足
   - ❌ 不适合：精密曲面、多实体装配约束

### 4.2 行业参考

180cm³ 的复杂零件，含 6 个特征组、30+ 体素操作、4 轮布尔运算、2 轮圆角，**一次 headless 脚本运行完成**。这在同等复杂度下，比 SolidWorks API 启动快约 3×（FreeCAD 启动 ~3s vs SW ~10s），且无需商业许可证。

### 4.3 遗憾与改进方向

| 维度 | 现状 | 理想方向 |
|------|------|----------|
| **圆角鲁棒性** | 失败时静默跳过 | 自动降级重试（R2→R1.5→R1） |
| **脚本复用** | 硬编码参数 | 抽离为参数 JSON + 通用生成器 |
| **验证自动化** | 仅体积自检 | 加 GD&T 形位公差检查 |
| **文档生成** | 无 | 工程图自动生成（FreeCAD TechDraw 脚本化） |
| **错误恢复** | 从头重跑 | 断点续跑（检查点机制） |

## 五、总结

> **一句话**：成功交付了 1 个有效实体、180cm³ 的工业法兰联轴器，几何准确度满足工程要求，主体圆角 100% 落地，螺栓孔细节圆角因 OCCT 内核限制部分缺失。

> **最大收获**：在自动化 CAD 管线中，**工序编排比算法强度更重要**——把圆角前置到拓扑最简单的时间窗口施，就绕过了内核的数学限制。

> **最大遗憾**：花了 30+ 次迭代才悟到这个道理。如果第一次就选择"先圆角后布尔"的策略，至少节省 80% 的调试时间。
